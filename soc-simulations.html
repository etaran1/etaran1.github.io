SOC Analyst Simulations

Two incident-response scenarios demonstrating alert triage, escalation logic, containment decisions, and documentation artifacts.

Phishing Email → Credential Attempt (Triage + Escalation)

Role Simulation: SOC Analyst (Tier 1–2)
SOC / Triage
Scenario

User reports a suspicious email; shortly after, identity provider logs show unusual sign-in attempts from a new location and device.

Analysis & Decisions
  • Validated the report and extracted indicators (sender domain, URLs, attachment hashes).
  • Reviewed IdP logs for risky sign-ins, new device enrollment, MFA failures, and token activity.
  • Correlated timing with email delivery to confirm likely credential capture attempt.
  • Escalated severity based on confirmed authentication anomalies.
  • Recommended containment: forced password reset, session/token revocation, and conditional access tightening.
Escalation rationale: Confirmed authentication anomalies + user-reported phishing indicates active credential compromise risk.
Outcome
  • Phish blocked across tenant; malicious sender domain and URLs added to deny lists.
  • Account secured with forced reset and session revocation; monitoring added for adjacent accounts.
  • Post-incident user guidance issued (reporting, MFA hygiene, suspicious link handling).
Artifacts Produced
Triage Notes IoC List (domains/URLs/hashes) Escalation Summary Containment Checklist User Advisory
Reflection
  • Token/session revocation is often faster than endpoint action for immediate containment.
  • Conditional access + risky sign-in policies reduce repeat incidents and speed triage.

Endpoint Malware Alert → Host Containment + Evidence Collection

Role Simulation: SOC Analyst (Tier 1–2)
SOC / IR
Scenario

EDR triggers on suspicious process behavior (possible credential dumping or persistence). Host also shows unusual outbound connections.

Analysis & Decisions
  • Confirmed alert fidelity: process tree, command line, parent/child relationships, and signed binary reputation.
  • Reviewed telemetry for C2 indicators: rare DNS, beacon-like intervals, and uncommon destinations.
  • Determined containment scope based on confidence and business impact (isolate host vs. block indicators).
  • Collected evidence for IR: timeline notes, key logs, hashes, and affected user and device context.
  • Escalated with a concise handoff package for Tier 2/IR investigation.
Containment choice: Isolate the host when behavior indicates credential access or persistence—limit blast radius first, then investigate deeper.
Outcome
  • Host isolated and indicators blocked; incident case opened with evidence bundle.
  • Follow-up monitoring placed on related endpoints, accounts, and network segments.
  • Recommended hardening actions: least privilege review, credential reset if needed, and control tuning.
Artifacts Produced
Incident Timeline EDR Evidence Summary Network IoCs Escalation + Handoff Notes Containment Decision Log
Reflection
  • Consistent handoff notes reduce time-to-containment for Tier 2/IR teams.
  • Process-tree context beats single-event alerts for confidence scoring and false-positive reduction.
